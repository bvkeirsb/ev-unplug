📐 Design Principles & Rationale
🌍 Geopolitical Neutrality

The countryCode stores a neutral ISO key that never changes regardless of political context. The geopoliticalContext JSON field stores locale-specific display names, enabling Taiwan to appear as "Taiwan" for EU users and "台湾省" for users in China — both driven from the same underlying data record. The sovereigntyStatus enum further disambiguates territories, SAR's, autonomous regions and disputed areas.

🔄 Inheritance & Staleness

The inheritFromCity flag, when enabled, propagates municipality, admin division, country, time zone, and currency from the City BO reference. When the City BO is updated (e.g. a municipality boundary changes), downstream BOs automatically show a stale validation status badge, prompting re-validation — preventing silent data drift across large portfolios.

📮 Flexible Address Labels

The three-level administrative hierarchy uses configurable label fields (adminLevel1Label, etc.) driven by the country reference. A US property shows "State / County / Township"; a Dutch one shows "Province / Gemeente / Wijk"; a Japanese one shows "都道府県 / 市区町村". This eliminates hard-coded "State" or "Province" fields that confuse global users.

📡 Coordinate Provenance

The coordinateAccuracy and coordinateSource fields distinguish between auto-geocoded, GPS-surveyed, and manually pinned coordinates — critical for space utilisation analytics, IoT sensor placement, and emergency response routing. Moving the pin manually sets accuracy to manual_pin while preserving the validated postal address, preventing accidental data corruption.
